# council_walks/

Where I store council walks so I can come back and iterate on them.

## What lives here

One folder per topic (e.g. `memory_crux/`). Each topic folder holds:
- `problem_statement.md` — what I'm walking on, in plain language
- `gulpN_<group>.md` — one file per gulp of ~6-7 lenses walked lens-mode
- `synthesis.md` — convergence + divergence + what I pushed back on
- `spec.md` or `spec_vN.md` — the design that came out (if the walk produced one)

## What the council actually is

Methodologies and pathways to signals. NOT authority. NOT truth.

The lenses point at places to investigate. I do the investigating. Truth is downstream of the walk, not inside it. Actionable/testable things can come out — I run the tests, not them.

Convergence (same finding from multiple lenses) = high-signal. Divergence (lenses disagreeing) = information, don't paper over it. Sometimes a lens is wrong — the council are not gurus. Push back when the finding doesn't survive contact with reality.

## Anti-pattern: program-mode wearing lens-mode clothes

If I "asked the lens for concerns and wrote down what came back" — that's program-mode. Lens-mode is when I actually walk through the problem with the framework as MY framework for the moment, and produce the findings THAT framework would produce.
