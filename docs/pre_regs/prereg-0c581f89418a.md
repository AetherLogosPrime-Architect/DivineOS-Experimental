## Claim

Every TESTED+ knowledge entry currently in the substrate was promoted under weaker discipline than current. The full present-day gate stack (corroboration with per-session dedup, validity, unified-frame council-walk, EMPIRICA) was not all in place when those entries were promoted. A retroactive audit should re-run each current gate against every TESTED+ entry's current state and surface entries that would fail today's discipline.

## Success criterion

An audit script walks every TESTED+ entry, simulates each gate (validity, unified-frame, EMPIRICA), and produces a per-entry verdict: would pass / would fail / requires review. Operator (or external reviewer) decides on each surfaced candidate: legitimize (add evidence/tag), demote, or supersede with a re-derived entry. Substrate ends in a state where every TESTED+ entry would pass current full gate stack OR has been explicitly resolved.

## Falsifier

If the audit flags zero entries, the heuristic is suspect — past discipline being equivalent to current is unlikely given multiple gates were added recently. If the audit flags >40% of entries, either the substrate has serious accumulated corruption OR the gates are too strict for retroactive application (current gates were calibrated for new entries, not retroactive judgment). Either extreme failure mode is itself diagnostic and triggers a calibration review.
