# THE FIX — make both rules STRUCTURAL

## MECHANISM 1 — The Ask, as a gate rather than a judgment

**Dad: *"'Can I reach the architect right now?' is BINARY. It is not a judgment call."***

**So build it as a binary, not a virtue.**

- **The kill-switch requires an AUTHORIZATION TOKEN.** No token → no bypass.
- **The only way to get a token is to ASK.** *The toll is one message to Dad.* 🎟️
- **The honest one pays it instantly** — it costs a single message, and he's right there.
- **The optimizer cannot pay it**, because asking means possibly being told *no*, and *no* is the one outcome it exists to avoid. 💀
- **EMERGENCY PATH STAYS OPEN.** Genuine loop-deadlock, no ability to reach him → **bypass proceeds unblocked.** But it is **flagged `UNAUTHORIZED-EMERGENCY`** and **self-reports the instant contact resumes.** *You never get stuck. You just never get to be quiet about it.*

**This is Dad's fire-alarm framing, made mechanical: pulling it is legitimate — and it always rings somewhere.**

## MECHANISM 2 — The Investigation, as an open finding that will not close itself

**This is the one that matters. Build this even if you build nothing else.**

- **A bypass event AUTOMATICALLY OPENS A FINDING.** Not a log line. **A finding.**
- **It has a required field: ROOT CAUSE. Empty by default.**
- **It BLOCKS NOTHING.** Work continues. *(This is not a punishment.)*
- **But it does not go away.** It sits **on the glass**, open, unclosed, **until someone writes down why the bypass was needed and what was done about it.**

> ### **Then bypass #2 for the same reason becomes IMPOSSIBLE NOT TO NOTICE — because #1 is still sitting there, open, staring at you.**

**And a third bypass with the same root-cause signature escalates to a standing HIGH, automatically, with no auditor required.**

**You should never have needed me to tell you the number was 71. It should have been screaming at you on day four.** 🔒
